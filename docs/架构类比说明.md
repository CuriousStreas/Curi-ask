# 架构类比：React+Express 与 Vue+Flask

## 一句话对应关系

| 角色     | 之前方案    | 现在方案   | 作用           |
|----------|-------------|------------|----------------|
| **前端** | React       | **Vue 3**  | 页面、交互、调后端 API |
| **后端** | Express     | **Flask**  | 提供接口、转发到 AI API |

前后端分离的思路完全一样：**前端只负责界面和发请求，后端负责鉴权、选模型、调你的聚合 API**。

---

## 前端：React 和 Vue 的类比

- **都是组件化**：页面由“组件”拼起来，数据变了界面自动更新。
- **React**：用 `useState` 存数据、`useEffect` 做副作用；JSX 里写 `{变量}`。
- **Vue**：用 `ref` / `reactive` 存数据，`onMounted` 等做“挂载后请求”；模板里写 `{{ 变量 }}` 和 `v-model`、`v-for`。

**对照表：**

| 要做的事           | React                     | Vue 3                          |
|--------------------|---------------------------|---------------------------------|
| 定义可变数据       | `useState(初始值)`        | `ref(初始值)` 或 `reactive({})` |
| 组件挂载后请求接口 | `useEffect(() => { ... }, [])` | `onMounted(() => { ... })`     |
| 双向绑定输入框     | `value={x}` + `onChange`   | `v-model="x"`                   |
| 列表渲染           | `arr.map(x => <div key=...>)` | `v-for="(x, i) in arr" :key="i"` |
| 引用 DOM           | `useRef(null)`             | `ref(null)`，模板里 `ref="名字"` |

所以：**逻辑一样，只是语法从 React 换成了 Vue**，你看到 Vue 的 `ref`、`onMounted`、`v-model` 就可以想到原来 React 里对应的那几样。

---

## 后端：Express 和 Flask 的类比

- **都是“一个进程监听端口，根据 URL 和方法调不同函数”**。
- **Express**：`app.get('/api/models', ...)`、`app.post('/api/chat', ...)`，用 `req.body`、`res.json()`。
- **Flask**：`@app.route('/api/models')`、`@app.route('/api/chat', methods=['POST'])`，用 `request.json`、`return jsonify(...)`。

**对照表：**

| 要做的事         | Express (Node)              | Flask (Python)                    |
|------------------|-----------------------------|------------------------------------|
| 定义 GET 接口    | `app.get('/path', (req, res) => { ... })` | `@app.route('/path')` 下写函数      |
| 定义 POST 接口   | `app.post('/path', async (req, res) => { ... })` | `@app.route('/path', methods=['POST'])` |
| 读请求体 JSON   | `req.body`                  | `request.get_json()` / `request.json` |
| 返回 JSON        | `res.json({ ... })`         | `return jsonify({ ... })`          |
| 返回错误码       | `res.status(400).json(...)` | `return jsonify(...), 400`         |
| 跨域             | `cors()` 中间件             | `flask_cors` 或响应头              |

所以：**接口设计不变（还是 `/api/models` 和 `/api/chat`），只是从 Express 的写法换成 Flask 的写法**。

---

## 整体流程（两种技术栈都一样）

1. 用户打开页面 → **Vue 前端**请求 `GET /api/models` → **Flask 后端**从配置里读模型列表 → 返回给前端展示在下拉框。
2. 用户选模型、输入问题、点发送 → **Vue 前端**把 `model` 和 `messages` 用 `POST /api/chat` 发给 **Flask 后端**。
3. **Flask 后端**根据 `model` 和配置的 `AI_API_BASE_URL` 去调你的聚合 API（OpenAI 兼容格式），拿到回复再 `jsonify` 给前端。
4. 前端把助手回复追加到对话里。

**优点（和用 React+Express 时一样）：**

- 前后端分离：前端可以单独部署、换域名；后端专注接口和密钥安全。
- 多模型：前端只传一个 `model` 字符串，后端统一决定调哪个 API、怎么调。
- API Key 只存在后端，浏览器里看不到，更安全。

你现在用的是 **Vue + Flask**，上面这些“优点”和“流程”完全适用，只是实现从 React/Express 换成了 Vue/Flask 的语法而已。
