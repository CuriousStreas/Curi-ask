# Nginx 网关与安全性

在 Curi Ask 前面加一层 **Nginx（或 Caddy、Traefik 等）反向代理**，可以明显提升安全性，推荐在生产环境使用。

## 能带来哪些安全提升

| 方面 | 说明 |
|------|------|
| **隐藏后端** | Flask 只监听 127.0.0.1:3001，不直接对外，外网只访问 Nginx。攻击面变小，且后端端口可不对外开放。 |
| **统一入口** | 前端静态 + `/api` 都走 80/443，便于做访问控制、限流、日志审计。 |
| **限流防刷** | 用 `limit_req` 限制同一 IP 的请求频率，防止接口被刷、滥用。 |
| **HTTPS** | 在 Nginx 上配 SSL 终止，用 Let’s Encrypt 等申请证书，全站 HTTPS，防窃听、防篡改。 |
| **请求过滤** | 可对请求头/体做校验、拦截异常大小或非法路径，减少直接打到后端的恶意请求。 |
| **日志与审计** | 在 Nginx 记录访问日志，便于排查问题、做简单安全审计。 |

## 使用方式概览

1. **开发**：直接跑 `start-backend.bat` + `start-frontend.bat`，无需 Nginx。
2. **生产**：
   - 前端：`npm run build`，把 `frontend/dist` 交给 Nginx 或任意静态托管。
   - 后端：Flask 只绑定 `127.0.0.1:3001`（本机），由 Nginx 反向代理到 `http://127.0.0.1:3001`。
   - Nginx：对外 80/443，`/` 走静态，`/api/` 走后端，并配置限流、HTTPS（推荐）。

项目里已提供 **`nginx/nginx.conf.example`**，复制后按实际路径和域名修改即可。

## 其他建议

- **API Key 只放后端**：`.env` 中的 `AI_API_KEY` 仅存在于服务器，浏览器永远拿不到，这是当前架构已经具备的安全点。
- **HTTPS**：生产务必用 Nginx（或 Caddy）做 SSL，可用 `certbot` 申请免费证书。
- **防火墙**：只开放 80/443，不要对公网开放 3001、5173 等端口。

结论：**增加 Nginx 网关能增加安全性**，建议在生产环境用 Nginx（或同类反向代理）做统一入口、限流和 HTTPS。
